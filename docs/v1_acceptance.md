# V1 Acceptance Checklist

## Mission Flow

- [ ] Start at city/type selection and enter 2D map.
- [ ] A generated call appears with limited caller information.
- [ ] Dispatching units enters moving loading-screen state.
- [ ] On first-arrival, session transitions into 3D mission scene.

## Gameplay Rules

- [ ] AI mission can inject dynamic escalations (flashover, victims, collapse, etc.).
- [ ] Incorrect/insufficient dispatch increases severity.
- [ ] Non-focused missions remain paused and do not escalate.

## UI/Controls

- [ ] Left panel shows selected on-scene units/personnel state.
- [ ] Right panel shows radio traffic messages.
- [ ] Bottom bar supports pause toggle.
- [ ] Easy mode provides Blue/Yellow/Red hinting.
- [ ] Normal/Hard hide color hinting and rely on reports/visuals.
- [ ] Left click acts as grab/select.
- [ ] Right click acts as contextual action.

## Modes

- [ ] Build mode supports credit progression and station purchases.
- [ ] Dispatcher mode starts with city real-life station capacity and no station buying.
